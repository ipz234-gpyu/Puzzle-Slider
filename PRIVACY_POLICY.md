# Privacy Policy, GDPR Compliance Guide, and Terms of Use for PuzzleSlider

**Effective Date:** 20 May 2026

## 1. Introduction and Project Scope

PuzzleSlider is a serverless, browser-based sliding puzzle game. The application is a client-side single-page app built with React 19, TypeScript, Vite, React Router DOM, Zustand, React Hook Form, and React Virtualized, organized under a Feature-Sliced Design structure. 

The application renders and executes entirely within the user's web browser. It does not integrate any backend API, database, account system, analytics SDK, advertising network, payment flow, or external data processor. No data entered or generated in this application is transmitted over the network to the developer or any third party.

The app uses locally bundled level images and metadata. User interaction is limited to gameplay, local configuration, consent management, and results display.

## 2. Data Processing (Local Only)

PuzzleSlider does not require or support user accounts. The only directly entered identifier is an optional player name. The application processes data strictly on the client side to provide interactive gameplay and save progress locally. 

The following data categories are processed **strictly within the user's browser runtime**:
1. **Player profile data:** The player name entered in the configuration form, used locally as the current player label.
2. **Consent data:** The user's storage preferences for game settings and puzzle progress.
3. **Gameplay data:** Completed level ID, grid size, elapsed time, number of moves, unlocked level size, sandbox status, and best records per grid size.
4. **Derived presentation data:** Results history, level names, and level images displayed from locally bundled assets.
5. **Technical state:** Loading flags, error flags, route parameters, and in-memory caches used to keep the app usable.

This application does not collect or transmit email addresses, passwords, phone numbers, payment information, geolocation data, precise device identifiers, or special-category data. No code path sends IP addresses, tracking cookies, or browser fingerprinting data to an external server. Any standard technical IP logging that occurs at the hosting infrastructure layer (e.g., GitHub Pages, Vercel, Netlify) is governed entirely by the respective hosting provider's privacy policy and is outside the control of this project.

## 3. Browser Storage Inventory

The application uses browser `localStorage` to persist game state across sessions. It does not use HTTP cookies or `sessionStorage`.

| Storage mechanism | Key or pattern | Data stored | Purpose | Consent and retention |
| --- | --- | --- | --- | --- |
| localStorage | cookie_consent_settings | Consent preferences with boolean choices for gameSettings and puzzle_progress | Remember and enforce the user's storage choices | Always saved so the app can remember consent state; retained until the user clears browser data. |
| localStorage | gameSettings | Current game settings, currently only the playerName field | Persist the player profile and related settings | Written only when the gameSettings category is allowed; deleted immediately when consent is withdrawn. |
| localStorage | puzzle_progress_<playerName> | Per-player puzzle progress, including isSandbox, currentUnlocked, and bestRecords with time and moves by grid size | Persist unlocked levels and best results for the active player | Written only when the puzzle_progress category is allowed; deleted immediately when consent is withdrawn. |
| In-memory only | cachedProgressByPlayer | Serialized progress cached in the Zustand store when progress persistence is not allowed | Keep the current session usable without writing progress to localStorage | Lost on refresh, tab close, private browsing reset, or process restart. |

*Note: The term "cookie" in the storage key name refers to a local user-interface convention for consent management and does not represent actual tracking or network cookies.*

## 4. GDPR Compliance (Client-Side)

### 4.1 Data Ownership and Controller Absence
Because this application runs entirely locally and does not transmit data to any remote server, **there is no central Data Controller**. The user maintains exclusive control over their data. The developer has no access to, and cannot review, modify, or delete, any data generated by the user within the application.

### 4.2 Legal Basis for Local Processing
The application operates on a consent-first model for optional local storage under GDPR compliance:

| Processing activity | Legal basis and rationale |
| --- | --- |
| Recording consent in localStorage | Necessary to remember and apply the user's privacy choices. |
| Persisting the player name in gameSettings | Explicit user consent for optional local storage of a profile field. |
| Persisting puzzle progress and best records | Explicit user consent for optional local storage of gameplay progress. |
| Temporary in-memory gameplay state and timers | Necessary to perform the core interactive gaming service requested by the user. |
| Loading bundled assets and local UI state | Necessary to render the web application interface. |

### 4.3 Data Subject Rights
Since all data is localized to the user's device, GDPR rights are exercised directly by the user through the application UI or browser controls:
1. **Access & Rectification:** The user can view and modify their player name and results directly in the configuration forms.
2. **Erasure (Right to be Forgotten):** The user can wipe all data instantly by changing their storage settings to "declined" within the app's consent panel or by clearing their browser's site data/cookies.
3. **Restriction & Objection:** The user can deny storage access at any time via the consent banner, forcing the app to operate in-memory only.
4. **Portability:** Data remains in the user's local browser storage environment. No remote data transfers occur.

### 4.4 Security and Limitations
* **No Encryption:** Data stored in `localStorage` is stored in plaintext by the browser. 
* **Environment Security:** `localStorage` relies on the browser's Same-Origin Policy. Users should ensure their browser environment and extensions are secure.
* **Data Volatility:** Local progress will be permanently lost if the user clears browser data, uses incognito/private mode, switches browsers, or changes devices. The application does not provide server-side backups or account recovery.
